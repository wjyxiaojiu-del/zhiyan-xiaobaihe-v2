## 三、具体优化方案

### 优化1：推断性统计分析 ⭐ 最高优先级

**当前痛点**：
- 用户只能计算Mean±SD±SE，无法判断组间差异是否显著
- 写论文需要t检验/方差分析结果，只能导出数据到SPSS/R重新做
- F15: 需要多重比较字母标记（a, b, c...）直接贴论文表格

**用户期望**：
上传数据 → 选分组列 → 一键得到带显著性标记的分析结果

**优化方向**：新增功能

**具体方案**：
1. 纯前端实现t检验（独立样本/配对）和单因素方差分析(ANOVA)
2. 多重比较：LSD法 + Duncan法（农学论文最常用），自动生成abc字母标记
3. 结果表格式：`组别 | 均值±SD | 字母标记`，可直接截图贴论文
4. 技术路径：JavaScript纯前端计算，不需要后端（t分布用近似算法，F分布查表或近似）
5. 新增一个Tab："📊 显著性分析"，放在实验计算区下方

**预期效果**：
- 用户不用导出到SPSS/R，在系统内完成"描述统计→显著性检验→字母标记"全流程
- 影响70%以上用户（做实验就要做统计检验）

---

### 优化2：数据处理全流程UX优化

**当前痛点**：
- F1: 确认导入无loading，用户不知道是卡了还是在处理
- F2: 宽表（10+列）浏览体验差，要左右滚
- F11: 不确定所有数据是否都成功导入
- F10: 错误提示只说"缺少标准曲线参数"，不引导去哪配
- F17: 多Sheet处理不直观

**用户期望**：
上传→清晰的进度反馈→直观的数据总览→明确的下一步指引

**优化方向**：体验优化

**具体方案**：
1. **Loading状态**：确认导入按钮点击后变灰+旋转动画+文字"正在导入..."
2. **宽表优化**：数据预览表添加"冻结首列"（样品名列固定，数值列可横滚）；列数>8时自动添加列筛选下拉
3. **导入确认**：导入完成后显示绿色横幅"✅ 已成功导入 48行 × 12列数据"
4. **智能引导**：错误/警告消息改为可点击链接（如"请先配置标准曲线 → 点此跳转"）
5. **多Sheet提示**：有多个Sheet时，在预览区上方显示"📋 检测到3个Sheet，当前显示Sheet1。点击切换 → [Sheet1] [Sheet2] [Sheet3]"

---

### 优化3：AI能力实质化

**当前痛点**：
- AI问答只是关键词匹配，没有真LLM调用
- D3: AI看不到用户上传的数据
- D1: 问到Protocol库外的内容就哑火
- D6: 聊天记录不保存

**用户期望**：
真正能理解问题的AI助手，懂我的数据、记得对话

**优化方向**：新增功能 + 体验优化

**具体方案**：
1. **AI后端升级**：将`/api/chat`从关键词匹配改为调用LLM API（复用已写好的`call_llm`函数和用户API Key体系）
2. **数据感知**：在AI分析功能基础上，让AI聊天也能读取当前导入的数据摘要作为上下文
3. **对话历史持久化**：将聊天记录存入`user_data`表，刷新不丢失，支持"清空对话"
4. **快捷入口**：在数据处理结果旁加"🤖 让AI解读结果"按钮，一键把统计结果发AI解读
5. **降级策略**：无API Key时自动用关键词匹配（保留当前能力），有API Key才启用真LLM

---

### 优化4：非线性标准曲线拟合

**当前痛点**：
- 只支持y=ax+b线性回归
- 很多实验（ELISA、酶动力学）需要二次多项式、Logistic四参数、指数曲线

**用户期望**：
选择拟合模型类型→自动拟合→输出方程+R²

**优化方向**：新增功能

**具体方案**：
1. 最小二乘法实现多项式回归（二次、三次）
2. Logistic四参数曲线：y = d + (a-d)/(1+(x/c)^b)，用Levenberg-Marquardt迭代
3. 拟合结果对比：同时显示2-3种模型的R²和AIC，推荐最佳
4. 技术路径：纯前端JS实现（已有线性回归基础，扩展即可）

---

### 优化5：移动端响应式适配

**当前痛点**：
- J1: 手机端字体小、按钮难点
- E1: 仪器热区SVG在手机上点不准
- I5: 实验基地无WiFi需要离线访问

**用户期望**：
手机上能流畅使用核心功能（查Protocol、看数据结果）

**优化方向**：体验优化

**具体方案**：
1. **响应式CSS**：添加完整的移动端断点（<768px），字体放大、按钮间距增加、表格横向滚动
2. **导航折叠**：移动端用汉堡菜单收纳10个导航链接
3. **PWA离线支持**：添加`manifest.json` + Service Worker，缓存Protocol内容、静态资源、计算器
4. **仪器页适配**：移动端热区SVG改为列表式交互（不用热区，直接点击名称展开步骤）
5. **优先适配页面**：首页（Protocol卡片）、Protocol详情页、数据处理结果页

---

### 优化6：团队协作功能

**当前痛点**：
- I1: 同课题组同学各自收藏Protocol，无法共享
- I2: 导师想看汇总的实验记录却看不到
- I3: 课题组自研Protocol不想公开

**优化方向**：新增功能

**具体方案**：
1. **课题组空间**：创建组、邀请成员（通过分享码）、设置角色（组长/成员/观察者）
2. **共享Protocol**：组内私有Protocol + 公共Protocol分离；支持"分享到课题组"
3. **实验日志聚合**：组长可查看组内所有成员的日志汇总
4. **数据表**: 新增`teams`、`team_members`、`team_protocols`表
5. **MVP范围**：先做创建组+邀请成员+共享收藏，后续迭代再加日志聚合和私有Protocol

---

### 优化7：计算器增强

**当前痛点**：
- C4: 每次配一样的Tris-HCl都要重新算
- C9: 计算历史不保存
- C2: 梯度稀释不灵活

**优化方向**：新增功能

**具体方案**：
1. **配方模板**：用户可保存"我的常用配方"（名称+试剂+浓度+体积），一键加载参数重算
2. **计算历史**：最近20条计算自动保存到`localStorage`，点击恢复
3. **自定义梯度**：支持输入"起始浓度、终止浓度、步数"或"手动输入各步浓度"
4. **关联Protocol配方**：计算器页面新增下拉"从Protocol加载配方"，选Protocol后自动填入试剂参数

---

### 优化8：多重比较字母标记

**当前痛点**：
- F15: 算完Mean±SD后还要手动去SPSS做Duncan多重比较，再手动标abc

**用户期望**：
算完方差分析后，直接在结果表里看到abc字母标记

**优化方向**：新增功能

**具体方案**：
1. 实现Duncan新复极差法和LSD最小显著差法（农学两大最常用法）
2. 输出格式：`均值 ± SD  字母标记` (如 `23.5 ± 2.1  a`)
3. 表格可直接复制到Word/Excel，字母标记保持
4. 与优化1（推断性统计）联动，ANOVA显著后自动触发多重比较

---

### 优化9：实验流程向导

**当前痛点**：
- A4: 65个Protocol都是独立的，真实实验往往需要串联
- 比如"可溶性蛋白测定"实际流程：取样→液氮研磨→称重→加提取液→离心→取上清→Bradford反应→测OD→算含量
- 这些步骤散布在多个Protocol里（P001研磨、P002 Bradford）

**用户期望**：
选择"我要测可溶性蛋白"→系统自动串联所有相关Protocol→给出完整流程

**优化方向**：新增功能

**具体方案**：
1. 预置5-8个常见实验流程模板（可溶性蛋白全流程、RNA提取全流程、酶活全流程等）
2. 每个模板是一个有序的步骤列表，每步链接一个Protocol的具体步骤
3. 流程图展示（类似Stepper组件），已完成步骤可勾选
4. 用户可自定义流程（拖拽Protocol卡片排序）

---

### 优化10：提示信息精度提升

**当前痛点**：
- F14: "OD值超出范围"没说是哪个样品哪一列
- F10: 缺少标准曲线参数只说"缺少"不引导去哪补
- G1: 提取缺失步骤不指出原文位置

**用户期望**：
错误提示精确定位 + 可操作的解决方案

**优化方向**：体验优化

**具体方案**：
1. **精确定位**：校验消息改为 `"第3行[样品A2]的A665列OD值(2.35)超出建议范围(0-1.5)，建议稀释5倍重测"`
2. **可操作链接**：`"缺少标准曲线斜率a → 点此跳转到标准曲线Tab"`
3. **分级展示**：错误(红, 阻断计算) / 警告(橙, 可继续但结果存疑) / 建议(蓝, 不影响计算的最佳实践)
4. **修复建议**：每个校验问题附带1-2条具体修复方案
